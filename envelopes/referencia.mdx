A API de envelopes permite criar, enviar, gerenciar e acompanhar documentos para assinatura digital, oferecendo funcionalidades completas para integrações server-to-server.

## Operações Disponíveis:

### Criação e Envio
- POST `/api/envelopes/send/base64` — enviar envelope com documento base64
- POST `/api/envelopes/send/template` — enviar envelope baseado em template

### Consulta e Listagem
- GET `/api/envelopes` — listar envelopes com filtros e paginação
- GET `/api/envelopes/{envelopeId}` — obter detalhes completos de um envelope
- GET `/api/envelopes/{envelopeId}/history` — obter histórico de ações do envelope

### Gerenciamento
- POST `/api/envelopes/resend` — reenviar envelope para destinatários
- POST `/api/envelopes/void` — cancelar envelope (ação irreversível)

## Características Principais:

### Criação de Envelopes
- **Upload direto**: Envie PDFs codificados em base64
- **Templates**: Use templates pré-configurados com sobrescritas
- **Validação automática**: Validação de PDF, campos e destinatários
- **Limite de tamanho**: Máximo 10MB por documento
- **Múltiplos destinatários**: Configure vários signatários e observadores
- **Campos personalizáveis**: Assinatura, texto, data, checkbox, radio
- **Envio automático**: Envelopes são enviados imediatamente após criação

### Gestão e Acompanhamento
- **Listagem paginada**: Com filtros por status, data e texto
- **Detalhes completos**: Acesso a todos os dados do envelope
- **Histórico de auditoria**: Rastreamento completo de todas as ações
- **Reenvio inteligente**: Apenas para destinatários pendentes
- **Cancelamento controlado**: Com justificativa obrigatória

### Status do Envelope
| Status | Descrição |
|--------|-----------|
| `draft` | Rascunho (não enviado) |
| `sent` | Enviado, aguardando assinaturas |
| `completed` | Todas as assinaturas coletadas |
| `voided` | Cancelado pelo remetente |
| `expired` | Expirado sem completar |

## Para quem é ideal:

- **Integrações server-to-server**: APIs robustas para sistemas corporativos
- **Automações de workflow**: Envio automático de contratos e documentos
- **Sistemas de gestão**: ERP, CRM e outras plataformas de negócio
- **Aplicações personalizadas**: Que precisam de controle total sobre o processo
- **Compliance e auditoria**: Com histórico completo e rastreabilidade

## Autenticação

Todas as operações requerem autenticação via Bearer Token:
```
Authorization: Bearer <access_token>
```

Obtenha seu token em `/api/token` usando suas credenciais de API Client.
